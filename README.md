# Scanner3D

Scanner3D - desktop-приложение на PySide6/PyQtGraph для 3D-сканирования измерительной области с управлением линейными осями, осью вращения, VNA и двумя генераторами сигналов.

Программа строит live-превью амплитуды и фазы, сохраняет полные измеренные данные, поддерживает sweep по координатам, sweep по частотам генераторов, rotation sweep, Z fly mode, центральную калибровку, импорт/экспорт JSON и отдельный time scan амплитуды/фазы от времени.

## Возможности

- Инициализация сканера по COM-портам: X, Y, Z и Rotation.
- Настройка масштаба осей: mm/step для X/Y/Z и deg/step для Rotation.
- Настройка скоростей, ускорений и торможений для каждой оси.
- Подключение VNA по TCP/IP.
- Подключение двух генераторов сигналов через Prologix/GPIB.
- 3D-измерение по осям X, Y, Z с возможностью отключать отдельные оси.
- Переключение плоскости отображения: `Z(X)`, `Y(X)`, `Z(Y)`, `X(Z)`, `X(Y)`, `Y(Z)`.
- Выбор slice по третьей оси и выбор rotation slice.
- Отображение амплитуды и фазы в реальном времени.
- Фаза отображается в градусах.
- Ручная и автоматическая настройка цветового диапазона графиков.
- Z snake mode для уменьшения лишних перемещений.
- Z fly mode для непрерывного движения по Z и измерения VNA на лету.
- Отметка late samples, если точка измерена после прохождения целевой координаты в fly mode.
- Центральная калибровка перед измерением и через заданное число полных Z-линий.
- Отображение raw или calibrated данных.
- Группировка сохраненных данных по частотам/амплитудам генераторов и rotation sweep.
- Отдельное окно `time_scan.py` для real-time графика амплитуды и фазы от времени.
- Сохранение полных данных в JSON с конвертацией NumPy-массивов только на этапе экспорта.
- Набор unit/integration/performance тестов с фейковыми данными и без обращения к реальному оборудованию.

## Требования

- Windows.
- Python 3.11-3.13.
- Доступ к COM-портам контроллеров осей.
- Доступ по сети к VNA.
- Доступ по сети к Prologix/GPIB адаптеру генераторов, если генераторы используются.

Python-зависимости указаны в [requirements.txt](requirements.txt):

- `PySide6`
- `pyqtgraph`
- `numpy`
- `PyOpenGL`
- `psutil`

В проекте также лежит библиотека `ximc` и `pyximc.py`, которые нужны для работы с контроллерами осей.

## Установка

Создать виртуальное окружение:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

Установить зависимости:

```powershell
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Если PowerShell не дает активировать окружение из-за execution policy, можно запускать Python напрямую:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe main.py
```

## Запуск

Основное приложение:

```powershell
python main.py
```

Или из виртуального окружения без активации:

```powershell
.\.venv\Scripts\python.exe main.py
```

Отдельный time scan:

```powershell
python time_scan.py
```

`time_scan.py` напрямую создает `VNABlock()` и применяет настройки из функции `configure_vna()`. Перед использованием проверь host/port VNA в коде или настрой адаптер VNA так, чтобы дефолтное подключение было корректным для стенда.

## Настройка оборудования

Все настройки сохраняются в `settings.ini` через `QSettings`. После успешной инициализации или запуска измерения приложение сохраняет актуальные параметры.

### Scanner Init

На вкладке `Setup` в блоке `Scanner Init` задаются:

- COM-порты осей `X`, `Y`, `Z`, `Rotation`.
- Масштаб `X/Y/Z`, mm/step.
- Масштаб `Rotation`, deg/step.
- Скорости, ускорения и торможения.

После заполнения нажать `Initialize`. Если подключение прошло успешно, статус покажет список инициализированных осей.

### VNA Init

В блоке `Vna init` задаются:

- `Host`
- `Port`

После заполнения нажать `Initialize`. Приложение создает `VNABlock`, выполняет проверочный запрос и сохраняет настройки при успешной инициализации.

### Generator Init

Для каждого генератора задаются:

- `Host`
- `Port`
- `GPIB`

Генераторы подключаются отдельно. Измерение может работать без генераторов, если sweep по генераторам не нужен, но при включенной частотной сетке ожидается корректная настройка генераторов.

## Измерение

Измерение настраивается на вкладке `Measure`.

### Оси

Для каждой оси `X`, `Y`, `Z` задаются:

- галочка включения sweep;
- `Start`;
- `Stop`;
- `Points`;
- `Step`.

Если галочка оси снята, эта ось не сканируется и используется как одиночная координата `0.0` в расчетной сетке данных. Для реального положения сканера в измерениях используются actual-координаты, которые возвращает контроллер.

### Rotation

Для вращения задаются:

- галочка `Rotation`;
- `Start`;
- `Stop`;
- `Points`;
- `Step`.

Если rotation sweep включен, данные сохраняются отдельными блоками по углам. При построении сохраненных данных приложение группирует блоки с одинаковыми частотами/амплитудами генераторов и открывает окно, где rotation можно переключать отдельным слайдером.

### Z Snake

`Z Snake` меняет направление прохода по Z между соседними линиями, чтобы не возвращаться каждый раз в начало Z-линии. Это уменьшает лишние перемещения.

### Z Fly

`Z Fly` включает непрерывное движение по Z с заданной скоростью. VNA измеряется во время движения. Для каждой точки сохраняется:

- целевой `z_request`;
- фактический `z_response`;
- задержка VNA `vna_latency_ms`;
- флаг `late_sample`.

`Auto Z fly speed` ограничивает скорость fly mode с учетом измеренной задержки VNA, чтобы уменьшить риск поздних точек.

### VNA

Поля VNA:

- `VNA points` - число точек sweep VNA.
- `VNA start time, s` - начало временного диапазона.
- `VNA stop time, s` - конец временного диапазона.
- `VNA bandwidth, Hz` - bandwidth.
- `VNA average` - включение усреднения.
- `VNA average count` - число усреднений.

Измерительная точка хранит усредненный комплексный ответ VNA: real/im, amplitude, phase и phase в градусах.

### Генераторы

Для каждого генератора задаются:

- `Generator start N, GHz`;
- `Generator stop N, GHz`;
- `Generator points N`;
- `Generator amps N`.

`Generator amps` задается списком через запятую, например:

```text
-20,-10,0
```

Число frequency points для генератора 1 и генератора 2 должно совпадать. Для каждой пары частот и амплитуд создается отдельный блок данных.

### Live preview

`Plot update, Hz` задает максимальную частоту обновления live-графиков. Данные измерения не теряются: измерительный поток заполняет NumPy-массивы, а GUI берет последний доступный снимок состояния и перерисовывает его с заданной частотой.

`Plot max pixels` управляет ограничением разрешения live-кадра:

- `Full resolution` или `0` - рисовать полное разрешение;
- значение больше `0` - ограничить количество пикселей только для live-отображения.

Полные измеренные данные при этом остаются в памяти и сохраняются без downsample.

## Центральная калибровка

Блок `Center calibration` включает калибровку по центральной точке.

Настройки:

- `Calibration X, mm`
- `Calibration Y, mm`
- `Calibration Z, mm`
- `Calibration period, Z lines`

Алгоритм:

1. Перед измерением выполняется измерение в центральной точке.
2. После заданного числа полностью просканированных Z-линий сканер возвращается в центральную точку.
3. В центре выполняется новое измерение.
4. Калибровочная точка сохраняется в `calibration_points` вместе с координатами и комплексным ответом.
5. Для строк между двумя калибровками рассчитывается поправка по изменению complex real/im относительно калибровочного вектора.

В интерфейсе есть галочка `Show calibrated data`. Если она включена и в данных есть калиброванные поля, графики отображают:

- `calibrated_amplitude`;
- `calibrated_phase_degrees`;
- `calibrated_complex_real`;
- `calibrated_complex_imag`.

Если калибровка выключена, в блоках данных хранятся только raw-поля.

## Визуализация

В основном окне отображаются два графика:

- Amplitude, dB.
- Phase, deg.

Доступные плоскости:

- `Z(X)`
- `Y(X)`
- `Z(Y)`
- `X(Z)`
- `X(Y)`
- `Y(Z)`

Слайдер slice автоматически переключается на третью ось, которая не входит в выбранную плоскость. Например, если строится `Z(X)`, слайдер управляет `Y`.

Для rotation sweep используется отдельный слайдер угла. Для данных с разными частотами/амплитудами генераторов окно строится отдельно для каждой группы частот/амплитуд, а rotation остается внутри окна.

Цветовой диапазон можно настроить руками через histogram/levels. Если пользователь двигает levels вручную, новые данные не сбрасывают ручной диапазон. Кнопка `Auto` возвращает автоматический расчет диапазона по текущим данным.

## Данные

Измерения хранятся в таблице на вкладке `Data`.

В таблице можно:

- добавить комментарий;
- сохранить выбранное измерение в JSON;
- построить графики через `Build`;
- удалить измерение из текущей сессии;
- импортировать JSON через `Import Data`.

При сохранении NumPy-массивы конвертируются в JSON-compatible списки только в момент экспорта. Это нужно, чтобы во время измерения и визуализации не тратить лишнюю память и время на постоянные преобразования.

Основные поля блока измерения:

- `freq_1`, `freq_2`;
- `amp_1`, `amp_2`;
- `rotation_angle`;
- `x`, `y`, `z`;
- `amplitude`;
- `phase`;
- `phase_degrees`;
- `complex_real`;
- `complex_imag`;
- `z_request`;
- `z_response`;
- `vna_latency_ms`;
- `late_sample`;
- `center_calibration`;
- `calibration_points`;
- `calibrated_*`, если включена центральная калибровка.

## Time Scan

`time_scan.py` предназначен для длительного измерения VNA в одной точке и отображения амплитуды/фазы от времени.

Особенности:

- real-time график амплитуды;
- real-time график фазы в градусах;
- unwrap фазы для удобного непрерывного отображения;
- настройка количества последних отображаемых точек;
- настройка диапазона `From point` / `To point`;
- все точки сохраняются, даже если на графике отображается только часть.

Запуск:

```powershell
python time_scan.py
```

По окончании или закрытии окна данные сохраняются в файл вида:

```text
meas_time_scanYYYY-MM-DD_HH-MM-SS.json
```

## Архитектура проекта

Проект постепенно вынесен из монолитного GUI-кода в слои:

```text
api/                      низкоуровневые адаптеры оборудования
application/measurement/  конфигурации, порты, planning, события измерений
application/visualization/ slicing и группировка данных для графиков
domain/measurement/       структуры данных, sample, calibration, data block
infrastructure/           Qt-адаптер измерительного потока и runtime из State
interface/                PySide6 GUI, виджеты, графики, таблица данных
store/                    состояние приложения, настройки, модели данных
tests/                    unit/integration/performance тесты
utils/                    совместимость и общие утилиты
```

Основная идея:

- domain не зависит от GUI;
- application описывает use case и контракты;
- infrastructure связывает use case с Qt и текущим `State`;
- interface отвечает за отображение и пользовательский ввод;
- тяжелые данные хранятся в NumPy-массивах, а не в Python-list по точкам.

Ключевые файлы:

- [main.py](main.py) - вход в основное приложение.
- [time_scan.py](time_scan.py) - отдельное окно time scan.
- [interface/measure_widget.py](interface/measure_widget.py) - UI настройки измерения.
- [infrastructure/measurement/qt_measure_thread.py](infrastructure/measurement/qt_measure_thread.py) - измерительный QThread.
- [domain/measurement/data_block.py](domain/measurement/data_block.py) - создание NumPy-блоков данных.
- [domain/measurement/calibration.py](domain/measurement/calibration.py) - расчет калибровочной поправки.
- [application/visualization/plot_slicing.py](application/visualization/plot_slicing.py) - извлечение 2D-срезов.
- [interface/plot_widgets.py](interface/plot_widgets.py) - PyQtGraph-графики.
- [store/data.py](store/data.py) - таблица измерений и JSON-сохранение.

## Тесты

Запустить все тесты:

```powershell
python -m unittest discover -s tests
```

Проверить компиляцию всех Python-файлов:

```powershell
python -m py_compile $(rg --files -g "*.py")
```

В PowerShell удобнее так:

```powershell
$files = rg --files -g "*.py"
python -m py_compile $files
```

Что покрыто тестами:

- создание measurement blocks;
- расчет и применение калибровки;
- slicing по разным плоскостям;
- группировка данных по частотам/амплитудам/rotation;
- live preview update coalescing;
- производительность больших тензоров;
- интеграционный сценарий измерительного потока с несколькими частотами генераторов;
- проверка, что последняя частотная точка не теряется и попадает в storage.

Performance-тесты используют фейковые данные и не обращаются к реальному оборудованию.

## Сборка exe

Для сборки используется [build.bat](build.bat):

```powershell
.\build.bat
```

Скрипт:

- выбирает `.venv\Scripts\python.exe`, если виртуальное окружение существует;
- проверяет, что `Scanner3D.exe` не запущен;
- сохраняет текущий `settings.ini`;
- устанавливает `pyinstaller`, если он отсутствует;
- добавляет в сборку `assets`, `ximc` и `pyximc.py`;
- удаляет временный `.spec`;
- кладет результат в:

```text
dist\Scanner3D\Scanner3D.exe
```

## Частые проблемы

### Не инициализируется ось

Проверить:

- правильность COM-порта;
- что порт не занят другой программой;
- питание контроллера;
- наличие `ximc/win64` и `pyximc.py`;
- корректность scale factor для оси.

### Не подключается VNA

Проверить:

- IP-адрес и порт;
- доступность прибора по сети;
- firewall;
- что VNA принимает SCPI/TCP команды;
- что выбран правильный формат канала.

### Не подключается генератор

Проверить:

- IP и порт Prologix/GPIB адаптера;
- GPIB address;
- сетевую доступность адаптера;
- что прибор включен и отвечает на команды.

### GUI тормозит при больших данных

Проверить:

- `Plot update, Hz`;
- `Plot max pixels`;
- включен ли overlay late samples;
- включено ли отображение calibrated data;
- размер тензора `Y x X x Z`;
- частоту ответов VNA.

Измерение и отображение разделены: измерительный поток заполняет данные, GUI забирает последний доступный кадр по таймеру. При слишком высокой частоте обновления GUI может пропускать устаревшие кадры отображения, но сами измеренные точки в data block не теряются.

### Цветовая шкала постоянно меняется

Если нужен стабильный диапазон, выставить levels вручную на histogram графика. После ручного изменения автообновление levels отключается для этого графика. Чтобы снова включить автоматический диапазон, нажать `Auto`.

## Рекомендации по работе

- Перед большим измерением сделать короткий тест на небольшой сетке.
- Проверить реальные направления осей и scale factor.
- Для fly mode сначала подобрать скорость на малой области.
- Для больших тензоров не ставить частоту live preview выше, чем реально нужно оператору.
- Сохранять измерение сразу после завершения, если данные важны.
- При изменении аппаратной конфигурации заново нажимать `Initialize` в соответствующем блоке.
